# Numbers 轉 BearHigh APP 切換清單

目標：把 Numbers 從日常操作中心降為唯讀歷史檔，之後新增收款、退班、堂次與老師薪資只在 BearHigh APP 處理。

## 真相來源

- 2026-07-22 正式切換時，使用者已明確指定目前開啟的 Numbers `高中部學生名冊.numbers` 為唯一匯入真相；先前已核對的學校、退班、重複列與誤勾修正均以這份目前狀態為準。
- 歷史學費、繳費日期、折扣、退費與已完成的老師月薪：以 2026-07-22 20:50 從目前 Numbers 匯出的 XLSX 快照為來源。
- 已進入 BearHigh 的收款、異動、堂次與月結快照：以 `bearhigh/accounting/manual/*` 為主。
- 切換完成後的新收款、退班、堂次與薪資只在 APP 操作；Google Sheet 與 Numbers 不再反向覆蓋 APP 新流水。

## 2026-07-22 正式切換結果

- 正式批次：`numbers-20260722-current`。
- 匯入 1,450 位學生、329 門課程、3,828 筆選課、3,665 筆應收、3,720 筆收退費流水、610 筆抵扣／退費／帳務備註、17 個歷史老師月薪快照。
- 逐科學收合計 NT$63,427,972，與 Numbers 非總計學收欄逐筆加總一致；退費合計 NT$1,181,300，與 Numbers 一致。
- 2026 年 5 月五位老師薪資合計 NT$1,022,990，逐人金額與 Numbers 一致。
- 舊 `masterImport` 資料採封存／作廢，不刪除；寫入使用 `/accounting` 下 20,977 個 scoped path 的單次原子 update，並關閉 Functions triggers。
- 寫入前完整備份保存在本機暫存區；正式 Firebase 已讀回批次指標、三位指定學生與 2026-05 薪資快照。

## 切換步驟

1. 建立切換快照
   - 從目前 Numbers 匯出一份 XLSX，不直接修改 `.numbers` 原檔。
   - 記錄匯出時間、來源檔與 import batch id。

2. 產生唯讀差異報告
   - Google Sheet 對 Numbers：學生 ID、姓名、學校、目前修課。
   - Numbers 對 BearHigh：應收、已收、繳費日期、折扣、退費、進退班、堂次與已完成月薪。
   - 分成「可自動新增」「內容相同」「需人工確認」「只能保留歷史」四類。

3. 先修正主檔，再搬流水
   - 先確認學生、老師、學期、課程與選課 id。
   - 再搬應收、付款、退費、異動與堂次。
   - 最後搬已完成月份的薪資快照，避免新舊公式混算。

4. 寫入安全邊界
   - 寫入前保存 dry-run JSON 與統計數字。
   - 只對 `bearhigh/accounting/manual/*/{id}` 做 scoped multi-location update。
   - 不整包覆蓋 `accounting` 或 `manual`，不刪除既有歷史紀錄。
   - 每筆保留來源 sheet、列號、欄名、原值與 batch id，方便追查。

5. 正式切換
   - 設定明確切換日期。
   - 切換日前的 Numbers 只保留查詢，不再新增日常紀錄。
   - 切換日後的收款、進退班、堂次與老師薪資只在 APP 操作。
   - 第一個完整月份結束後，比對 APP 月結與 Numbers 歷史公式；一致後才宣布 Numbers 退場。

## 後續再次匯入的停止條件

- 同名或重複學生尚未以永久學生 ID 解決。
- Google Sheet 課程名單與 Numbers 有差異但尚未確認。
- 學費總額正確、各科分攤不明。
- 退班日期、已上堂數或特殊人工折扣只能從備註猜測。
- 老師薪資規則或某月堂次資料不完整。

以上任一情況不得猜值；保留 raw snapshot 與明確備註，待使用者確認後才重跑正式匯入。
