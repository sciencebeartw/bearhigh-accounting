# BearHigh 高中帳務 APP 實際操作教學

最後更新：2026-07-22

這份教學完全依照目前的工作流程排列。日常操作以 APP 為主，Numbers 保留作為歷史查詢；Google Calendar 與行政老師點名單目前仍是堂次與到班人數的核對來源。

## 1. 開始使用

1. 開啟正式網址：`https://www.sciencebear.com.tw/bearhigh-accounting/`
2. 用已授權的 Google 帳號登入。
3. 登入後先停留在「工作台」，確認頁面有顯示學生、課程與月結資料。
4. 日常主選單只需要記住四個入口：「工作台」、「學生帳務」、「課程與名單」、「老師薪資」。進退班、詳細堂次與帳務工具可從工作台捷徑進入。

### 第一次正式測試

先只做唯讀核對，不要建立虛構學生或虛構收款：

1. 在「學生帳務」查 2–3 位熟悉的學生，看學校、修課、學收與收款是否正確。
2. 在「課程與名單」查 2–3 個班，看學生名單與老師是否正確。
3. 在「老師薪資」查已存月結：2026 年 5 月總額應為 `1,022,990`，6 月總額應為 `699,530`。
4. 第一筆寫入請等真實的入班、退班、換班或堂次發生時再做。目前不把假資料當成測試資料寫進正式帳務。

## 2. 新生入班或舊生加科

點選「工作台」→「記進班／退班」，開啟「新生／加科入班」。

1. 先在「已有學生」搜尋。搜得到就選原學生；只有確定不存在時才選「＋新增學生」。
2. 填寫姓名、分頁／年級、高中／學校、年級文字與家長手機。
3. 選擇「課程／科目」。
4. 「這一科學費」要填這科實際分攤的收入；合報時不要每科都填 21,600。
5. 「學生第一堂」填該生真正開始計入這班人數的日期。入班日當堂就會加一人。
6. 有收款就填「匯款／繳費日」與收款方式；尚未收款可先留空，之後再處理收款。
7. 合報、分期或特殊狀況寫在備註。
8. 先看表單下方摘要，確認學生、課程、金額與日期後，按「確認入班並同步老師名單」。

完成後應同時看到：

- 學生已出現在該課程名單。
- 學生帳務有這科應收，有填繳費日時也有收款流水。
- 進退班紀錄有入班日。
- 下次算薪資時，從學生第一堂起人數加一。

若同一位學生一次報多科，每一科各做一次入班，並填該科實際分攤金額與開始日。

## 3. 收款、合報、分期與抵用券

在「學生帳務」先選學生，再點收款工具，或由「工作台」→「記收款／退費」進入「學生收款」。

1. 選學生與本次收款科目。
2. 填收款日期、本次實收、付款方式。
3. 兩科合報優惠填 `1,000`，三科填 `3,000`，四科填 `5,000`；另有抵用券就單獨填「抵用券」。
4. 分期時，「本次實收」只填這次真正收到的金額，並在「分期備註」寫第一期、尾款等。
5. 先看預覽中每科分攤與剩餘未收，確認後按「分攤並記入收款」。

完成後，「目前收款狀態」應顯示每科已收與未收，「收款／退費紀錄」應有這筆實際金流。

## 4. 舊生退班與退費

在「學生帳務」先找到正確學生，再進入「退班退費」。

1. 選擇要退的「應收／科目」。
2. 對照 Google Calendar 與點名單，確認已上堂數。
3. 「退班日期」是第一堂不再計入人數的日期。例如 6/15 退班，6/15 當堂就不算這位學生。
4. 「從第幾堂退」與「已上堂數」應相符；有完整堂次日期時，優先以日期核對。
5. 中途退班的已上堂數預設每堂 `1,000`元。
6. 若原本合報，APP 會依退班後剩餘科目數重算：新制 `21,600 / 42,200 / 61,800 / 81,400`；舊制 `16,800 / 32,600 / 47,400 / 62,200`。
7. 高三學測／分科若沒有唯一可確定的套餐價，不要猜。手動填「退班後剩餘科目學費」，核對無誤才繼續。
8. 檢查預覽：`退費＝已收總額－剩餘科目重定價－已上堂數 × 1,000`。
9. 選退費方式、寫備註，確認後按「記入退班退費」。

完成後應同時看到退費流水、選課狀態、退班事件與稽核紀錄已更新；之後的該班堂次人數會自動少一人。

## 5. 同科正式換班或暫時換班

點選「工作台」→「記進班／退班」，開啟「同科換班」。

1. 選學生、原班、新班。系統只允許同科、同學期換班。
2. 填「生效日」：當天起原班少一人，新班加一人。
3. 正式換班選「正式換班」。只是因進度暫時調整，選「暫時換班／後續換回」並填換回日。
4. 寫原因，例如「學校進度調整」。
5. 先看摘要，確認老師、日期與方向後，按「確認換班，同步兩班計薪人數」。

換班不新增學費、不退費、不重算合報，也不改寫已存的歷史薪資快照。實際授課老師與薪資歸屬老師可以不同，薪資以課程主檔的「薪資歸屬老師」為準。

## 6. 每月老師薪資

建議月底先開著 Google Calendar、點名單與 APP，逐班核對。

### A. 先確認每班堂數與日期

1. 在「工作台」選「月結月份」。
2. 看「班級堂次核對」與「月結作業清單」，先處理顯示缺堂次的班。
3. 點「補堂次」或「老師薪資」→「補堂次／詳細計算」。
4. 選月份與老師名單區塊，將 Google Calendar 與點名單都確認過的上課日期一行填一堂。
5. 例如：

   ```text
   2026-07-04
   2026-07-11
   2026-07-18 | 不收費
   2026-07-25
   ```

6. 「不收費」、補課或其他特殊文字要保留在日期後方。APP 會標成請覆核，不會自己猜這堂是否付老師薪資。
7. 堂數與日期數量一致後，按「儲存堂次日期」。

### B. 核對每堂人數

1. 看堂次日期下方的「計薪人數」預覽。
2. 逐堂對照點名單，檢查名單人數、當堂加入與當堂退出。
3. 已在 APP 做過新生入班、退班或換班的學生會自動出現，不用再手動加減。
4. 若是舊資料漏記，才在「本班本月進退班」補學生、加入／退出、日期或第幾堂，再按「加入本班異動」。
5. 規則固定為：入班日當堂計入；退班日當堂不計入。

### C. 試算、月結與薪資條

1. 點「預覽薪資試算」，確認該班逐堂人數、公式與小計。
2. 如果要依班完成草表，按「把這個班加入本月薪資表」。
3. 所有班都補齊後，回到「月底薪資結算」，點「產生月底結算」，或在工作台點「產生／重算本月薪資」。
4. 確認每位薪資歸屬老師的課程數、堂數／時數、人次、應付小計與異動備註。
5. 學測生物實際授課是許昱、學測地科實際授課是昱維，但薪資歸在周逸老師，這是正確設定。
6. 115 學年度高一竹中數學由黃浩老師授課，高二竹中數學由明軒老師授課。
7. 檢查沒有「缺堂次」、「請覆核」、不明老師或特殊套餐後，才點「確認並儲存快照」／「儲存月結快照」。
8. 儲存後再點「列印 / 存 PDF」，交給老闆或老師。要列某位老師的單獨薪資條，用老師總表該列的操作按鈕。

已存月結是歷史快照。之後改名單、堂次或公式，不會自動改掉已存的五月、六月或其他月份。

## 7. 修改課程的授課老師與薪資歸屬

在「工具與設定」→「課程」查找課程。

- 「實際授課老師」是站在教室上課的人。
- 「薪資歸屬老師」是月結上應付款的人。
- 這兩個欄位可以不同，不要為了顯示授課老師而改掉薪資歸屬。

在「工具與設定」→「老師」可設老師預設計薪模式。修改公式只影響尚未儲存的新結算，已存歷史快照不會回算。

## 8. 哪些情況要停下來再確認

遇到以下情況，不要猜、不要連續再建更多紀錄：

- 同名學生無法確定是哪一位。
- 課程、學期或老師對應不唯一。
- Google Calendar 與點名單的日期不同。
- 高三學測／分科的剩餘科目學費無法從已確認價格唯一推回。
- 特殊堂次是否要付老師薪資不明。
- 剛剛按錯、日期或金額填錯。

如果剛剛寫錯，先記下「學生、課程、日期、按了哪個按鈕」，不要用第二筆假紀錄互相抵銷。請 Codex 先做單筆 scoped read，確認關聯路徑後再修正，避免影響其他學生或已存薪資。

## 9. 每次完成後的 30 秒檢查

1. 學生對不對。
2. 課程與學期對不對。
3. 生效日是否是第一堂要加入／不計入的日期。
4. 學費、實收、退費是否與預覽一致。
5. 課程名單與進退班記錄是否同時更新。
6. 月底才儲存薪資快照；快照儲存前必須沒有尚未解決的警示。
